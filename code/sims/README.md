# Numerical experiments in the `SurvNPSA` paper

This directory contains the R code and processed simulation results for the paper "Nonparametric Sensitivity Analysis for Unobserved Confounding with Survival Outcomes".

Run all commands below from this directory.

------------------------------------------------------------------------

## Directory structure

-   `sim_main.R`\
    Main script for running one simulation replication.

-   `submit_sim.sh`\
    Shell script for running `sim_main.R` locally or on a Slurm cluster.

-   `sim_results.R`\
    Script for generating the simulation figures from the processed simulation results.

-   `compute_data/`\
    Seed data and true target parameters used in the simulations.

-   `output.paper/`\
    Processed results for the original simulation using the single GAM-based nuisance estimators.

-   `output.paper.superlearner/`\
    Processed results for the additional simulation using SuperLearner nuisance estimators.

-   `figures/`\
    Simulation figures generated by `sim_results.R`.

-   `utils/`\
    Helper functions used in the simulations.

------------------------------------------------------------------------

## R packages

The simulation scripts use `dplyr`, `SuperLearner`, `survSuperLearner`, `mgcv`, and `mvtnorm`. The additional SuperLearner simulation also requires packages used by the candidate learners, including `ranger`, `earth`, `xgboost`, and `randomForestSRC`. The result-processing script uses `patchwork`, `tidyr`, `dplyr`, and `ggplot2`.

------------------------------------------------------------------------

## Running simulation replications

The supplied seed file contains repetitions `j = 501, ..., 1500` for sample sizes `n = 500, 1000, 2500, 5000`.

To run a single replication from the original simulation:

```bash
bash submit_sim.sh 501 1000 gam
```

To run a single replication from the additional SuperLearner simulation:

```bash
bash submit_sim.sh 501 1000 superlearner
```

If the third argument is omitted, the original `gam` simulation is used by default:

```bash
bash submit_sim.sh 501 1000
```

You can also submit simulations to a Slurm cluster:

```bash
sbatch --array=501-1500%100 submit_sim.sh 1000 gam
sbatch --array=501-1500%100 submit_sim.sh 1000 superlearner
```

Outputs from the original simulation are saved in `output.paper/`. Outputs from the additional SuperLearner simulation are saved in `output.paper.superlearner/`.

------------------------------------------------------------------------

## Generating figures

The combined simulation results used to generate the submitted figures are provided as:

-   `output.paper/sim.res.paper.gam.RData`
-   `output.paper.superlearner/sim.res.paper.SL.RData`

The commented code at the top of `sim_results.R` shows how the individual simulation output files were combined into these data files. To generate the figures from the provided combined results, run:

```bash
Rscript sim_results.R gam
Rscript sim_results.R superlearner
Rscript sim_results.R both
```

These commands generate:

-   `figures/Fig_sim_bias.png` and `figures/Fig_sim_CI.png` for the original simulation;
-   `figures/Fig_sim_bias_SL.png` and `figures/Fig_sim_CI_SL.png` for the additional SuperLearner simulation;
-   `figures/Fig_sim_SL_sd.png` comparing Monte Carlo standard errors from the two simulations.

------------------------------------------------------------------------

## Reproducibility note

The seed file fixes the simulated datasets used in the numerical experiments. Some nuisance estimators, especially GAM, SuperLearner, survival learners, and `xgboost`, may produce small numerical differences across R versions, package versions, operating systems, or BLAS/LAPACK libraries. Therefore, rerunning individual replications in a different computing environment may give results that differ slightly from the processed results provided here. The combined `.RData` files in `output.paper/` and `output.paper.superlearner/` are the processed simulation results used to generate the submitted figures.
